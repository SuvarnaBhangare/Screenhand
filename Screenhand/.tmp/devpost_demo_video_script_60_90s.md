# ScreenHand Devpost Demo Video Script (60-90s)

Goal: maximize judge clarity in first 20 seconds, then prove real workflow execution.

## Version A (75s)

### 0-6s: Hook
Visual:
- Title card + quick desktop scene
Voiceover:
- "Most AI agents can reason, but they still can’t reliably execute real desktop workflows."

### 6-15s: Product Positioning
Visual:
- Show ScreenHand logo + "MCP Desktop Automation"
- Overlay: "macOS + Windows"
Voiceover:
- "ScreenHand is an open-source MCP server that gives AI agents eyes and hands across desktop apps and browser tabs."

### 15-30s: Capability Proof
Visual:
- Agent inspects UI tree
- OCR snapshot on desktop window
- Click/type action in native app
Voiceover:
- "It can read UI structure, run OCR, click, type, and control app elements using OS-native automation APIs."

### 30-50s: End-to-End Workflow
Visual:
- Multi-step task: read data in one app -> open Chrome -> submit form -> copy result to another app
- Show step overlays: 1/2/3/4
Voiceover:
- "Here the agent completes a full cross-app workflow: extract, navigate, fill, verify, and hand off results."

### 50-63s: Reliability + Technical Edge
Visual:
- Tool categories panel
- Architecture mini-diagram (MCP runtime + native bridge)
Voiceover:
- "Unlike browser-only bots, ScreenHand unifies native desktop control and browser automation in one MCP tool layer."

### 63-72s: Use Cases + Impact
Visual:
- Cards: QA, onboarding, support ops, growth ops
Voiceover:
- "Teams use ScreenHand for QA regression checks, onboarding automation, and repetitive operations."

### 72-75s: CTA
Visual:
- GitHub + Devpost URL + "Try it"
Voiceover:
- "Try ScreenHand on GitHub and run your first desktop automation workflow today."

## On-screen Text Pack (copy-paste)
- "AI Agent Desktop Automation via MCP"
- "See • Click • Type • Automate"
- "Native Apps + Browser"
- "macOS + Windows"
- "25+ MCP Tools"
- "Cross-App Workflow Execution"

## Recording Checklist
1. Keep cursor movement deliberate and slow enough for judges to follow.
2. Use one concrete task with visible before/after result.
3. Use zoom/crop so action areas are readable on mobile.
4. Keep subtitles always on.
5. Show final success state for at least 2 seconds.

## Export Spec
- Aspect: 16:9
- Resolution: 1920x1080
- Duration target: 70-80s
- Format: MP4 (H.264)
- Loudness: clear voiceover, low background music
