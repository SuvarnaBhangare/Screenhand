## 3) AEO / GEO Layer for AI Citation

Include FAQ block in project story with direct Q/A:
- What is ScreenHand?
- How is it different from browser-only tools?
- Which AI clients support it?
- What real workflows does it automate?

Writing rules for AI retrieval:
- Use answer-first paragraph under each section heading.
- Keep paragraphs short (2-3 sentences).
- Add concrete nouns and technical entities (MCP, CDP, Accessibility API, OCR).
- Add specific numbers where available (tool count, key capabilities).

External authority linking in story:
- Link MCP official site once.
- Link GitHub repo and npm package.
- Link docs/website with setup steps.

## 4) Devpost-Specific Distribution Strategy

Submission conversion:
- Publish as complete as possible on first live version.
- Add a clear thumbnail + demo video because project browsing is visual-first.
- Keep headline and first 2 lines extremely scannable for feed views.

Engagement loop (first 7 days):
- Day 0: publish + share on X/LinkedIn/Reddit with Devpost URL.
- Day 1-2: update project with one concrete improvement (new screenshot or workflow demo).
- Day 3-4: post a technical walkthrough thread and link back.
- Day 5-7: ship one case-study update section in story.

Hackathon strategy:
- Submit to hackathons where categories match: devtools, AI tooling, productivity, automation.
- For each submission, customize intro paragraph to hackathon theme/judging criteria.
- Keep one reusable base story + one tailored variant per hackathon.

## 5) Weekly Maintenance Cadence

Every week:
1. Refresh first paragraph with latest capability or proof point.
2. Add one new image/gif from real workflow.
3. Add one FAQ entry from user questions.
4. Verify all links and assets still load.
5. Re-check tag relevance based on new feature scope.

## 6) KPI Tracking (simple)

Track weekly:
- Profile visits from Devpost
- Project views
- Likes/comments
- Click-throughs to GitHub/npm/docs
- Number of hackathon submissions using this project page

Interpretation:
- High views + low clicks: improve tagline + first paragraph + thumbnail.
- High clicks + low likes/comments: strengthen social proof and demo clarity.
- Low views: improve distribution cadence and hackathon placement.
